# Understanding the BOQ

This guide explains how to read and use the Bill of Quantities generated by Metraj.

---

## BOQ Structure

The generated BOQ is organized into sections that follow standard construction trade divisions. Each section groups related materials together, and within each section, individual line items list the specific materials with their quantities.

### Section Order

Sections appear in the following order (empty sections are omitted):

1. **Substructure** -- Foundations, ground slabs, basement materials
2. **Structural Frame (Columns & Beams)** -- Concrete, reinforcement, and formwork for columns and beams
3. **External Walls** -- Structural materials, external finishes, and weatherproofing for outer walls
4. **Internal Walls & Partitions** -- Brick, block, or concrete for interior walls plus internal finishes
5. **Upper Floor Slabs** -- Concrete, reinforcement, formwork, and finishes for floor slabs
6. **Roof** -- Roof structure and covering materials
7. **Doors** -- Door units, frames, and hardware
8. **Windows** -- Window units and sills
9. **Stairs & Ramps** -- Staircase materials
10. **Finishes** -- Floor, wall, and ceiling finishing materials
11. **MEP** -- Mechanical, electrical, and plumbing (if modeled)
12. **External Works** -- Landscaping and site materials (if modeled)

## Excel Report Structure

The Excel report contains three sheets:

### Sheet 1: Bill of Quantities

The main BOQ sheet with a professional layout:

**Header area:**
- Title: "BILL OF QUANTITIES" (or equivalent in Turkish/Arabic)
- Project name (from the IFC file's IfcProject entity)
- Building name (if available)
- Date and "Prepared by: Metraj AI System"

**Column headers:**

| Column | Header | Description |
|---|---|---|
| A | Item No. | Hierarchical item number (e.g., "2.03" = Section 2, Item 3) |
| B | Description | Material name and specification |
| C | Unit | Measurement unit (m2, m3, m, kg, nr, set) |
| D | Quantity | Total quantity including waste factor |
| E | Rate | Unit price -- left blank for user to fill in |
| F | Amount | Total cost -- formula: Quantity x Rate |

**Section rows:** Each section has a shaded header row with the section name, followed by its line items, followed by a subtotal row with a SUM formula for column F.

**Grand total:** At the bottom, a grand total row sums all section subtotals.

**Sample BOQ snippet:**

### Sheet 2: Material Summary

A flat list of all materials across all sections:

| Column | Header | Description |
|---|---|---|
| A | No. | Sequential number |
| B | Material | Material description |
| C | Unit | Measurement unit |
| D | Total Qty | Total quantity including waste |
| E | Category | Which BOQ section this material belongs to |

This sheet is useful for procurement -- it gives a quick overview of all materials to be ordered.

### Sheet 3: Audit Trail

The audit trail provides traceability for every BOQ line item:

| Column | Header | Description |
|---|---|---|
| A | Item No. | Same item number as Sheet 1 |
| B | Description | Material description |
| C | Unit | Measurement unit |
| D | Base Qty | Net quantity before waste factor |
| E | Waste % | Waste factor applied (e.g., 5%, 10%) |
| F | Total Qty (incl. waste) | Final quantity after waste: Base Qty x (1 + Waste %) |
| G | Source Elements | Number of IFC elements that contributed to this item |

The audit trail allows a quantity surveyor to:
- Verify how waste was applied to each item
- Understand the difference between base and total quantities
- See how many building elements were aggregated into each line item
- Trace quantities back to their sources

## Number Formatting

Quantities are formatted based on their unit type for readability:

| Unit | Format | Example |
|---|---|---|
| m2 (square metres) | 2 decimal places | 138.60 |
| m3 (cubic metres) | 3 decimal places | 45.200 |
| m (linear metres) | 2 decimal places | 25.50 |
| kg (kilograms) | 1 decimal place | 4,520.0 |
| nr (number/count) | No decimals | 15 |
| set | No decimals | 15 |

## How Waste Is Included

Every quantity in the BOQ already includes the waste factor. The "Quantity" column on Sheet 1 shows the **total quantity** (base + waste), which is the amount to actually order.

To see the breakdown:
1. Open the **Audit Trail** sheet (Sheet 3)
2. Compare the "Base Qty" column (net requirement) with the "Total Qty" column (with waste)
3. The "Waste %" column shows the factor applied

For example, if a wall requires 100 m2 of internal plaster:
- Base Qty: 100.00 m2
- Waste %: 10%
- Total Qty: 110.00 m2 (this is what appears in the main BOQ)

## How to Use the BOQ for Procurement

### Step 1: Review the Material Summary

Start with Sheet 2 (Material Summary) to get an overview of all materials needed. This gives you the shopping list at a glance.

### Step 2: Add Unit Prices

In Sheet 1 (BOQ), fill in the **Rate** column (E) with current unit prices from your suppliers. The **Amount** column (F) will calculate automatically using the formula `Quantity x Rate`.

### Step 3: Review Section Subtotals

Each section has a subtotal row. These automatically sum the amounts for all items in that section, giving you cost breakdowns by trade.

### Step 4: Check the Grand Total

The grand total at the bottom sums all section subtotals, giving you the estimated total material cost.

### Step 5: Verify Against Audit Trail

For any line item that seems unusual, check Sheet 3 (Audit Trail) to understand:
- How many elements contributed to that quantity
- What waste factor was applied
- What the base quantity was before waste

### Step 6: Adjust for Local Conditions

Consider adjusting quantities for:
- Local waste rates that differ from the defaults
- Materials not covered by the IFC model (MEP, site works, temporary works)
- Specification changes (different concrete grade, different brick type)

## Language Support

The BOQ is generated in the language selected during upload:

| Language | Section titles example | Material names |
|---|---|---|
| English | "External Walls", "Structural Frame" | "Concrete C25/30", "Reinforcement steel" |
| Turkish | "Dis Duvarlar", "Tasiyici Sistem" | "Beton C25/30", "Donati Celigi B500" |
| Arabic | "الجدران الخارجية", "الهيكل الإنشائي" | "خرسانة C25/30", "حديد تسليح B500" |

Arabic reports are generated with right-to-left (RTL) sheet orientation.

To regenerate the BOQ in a different language, use the reprocess endpoint without re-uploading the file.

## CSV and JSON Formats

### CSV

A flat file with one row per BOQ line item, including section name. Suitable for importing into other spreadsheets, databases, or cost estimation software.

Columns: Item No., Description, Unit, Quantity, Base Qty, Waste %, Rate, Amount, Section

### JSON

A structured representation of the full BOQ data, including all sections, items, project metadata, and validation results. Suitable for programmatic access, integration with other systems, or custom reporting.
